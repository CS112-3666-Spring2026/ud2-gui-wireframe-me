My idea for the project is a tower defense game, tentatively called Castle Tower defense at the moment. (It very well may change, but I like 
castles so probably not). It's going to be pretty basic, where enemies move along a line towards the castle.
I'll use inheritance in the enemy class, or the Tower one. For example, I might have a fast enemy, or a slow one. They'll still be 
enemies, just with diffferent attributes. Same thing for towers. I'll have a light tower, like archers, then a heavy one, like a cannon.
I'm implementing polymophism through the Wave class, because that class will store different enemies in an array.
An abstact class in this project would be Tower, because there is no "Tower" in the game. It'll be either an archer or cannon.
I won't be using inner classes because all the things I have are distinct and important enough to be separate.
I won't use custom exception because all I need is simple validation. If something is negative that shouldn't be, like speed or health,
it'll just be set to 0.
